SAP HANA Configuration

This page defines the parameters specific for SAP HANA installations.

Virtual IP Address
is the IP address that the current SAP HANA primary instance has.
Replication mode
is the system replication mode between the two HANA systems. Please refer to the HANA Administration Guide for details.
Prefer Site Takeover
defines, if the secondary instance should take over, if the primary instance fails. If "false", prefer local restart of SAP HANA.
Automatic Registration
defines, if the former primary should be registered as the new secondary after the takeover procedure.
Perform backup
defines, if the primary instance should create a backup before enabling system replication. If this is unchecked, it is assumed that the backup was already created manually. If backup is not created during this step, the Wizard will fail to register secondary HANA system. In order to perform initial backup, the Wizard needs to have a key in Secure User Storage. If you are configuring a multi-tenant HANA installation (default for HANA versions 2.0 SPS01 and above), please make sure that the secure store key is properly configured for connecting to the SystemDB, i.e., specify port 3<SID>13 in the environment settings and SystemDB as the target database.

Note that the wizard can only perform backup if the secure user store has been configured prior to running the wizard.
